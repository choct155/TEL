Panel Analysis Example¶

This script serves as an example/playground for panel data operations. Initially, we will explore how pandas structures panel data. The brief discussion in McKinney's book reads in panel data from an external source (Yahoo financial data), using the following syntax: >panel_data=pd.Panel(dict((x,web.get_data_fred(x,start_date,end_date)) for x in x_list))

In this formulation, we have pulled data from the St. Louis Fed instead of Yahoo financial. The x represents an indicator, and x_list represents ... a list of indicators. We will unpack this call so that we can abstract to a more general formulation for panel data.

In the first cell, we simply get a sense of how much work the FRED pull is doing in this scheme.

This is where things get interesting. pd.Panel actually takes a dictionary argument, with each key corresponding to a different group in the panel. The dictionary constructor used in the McKinney example is actually a little unusual. It relies on the fact that dictionaries can be built from tuples.

A brief example will show what I mean. We will construct the following dictionary: >ex_dict={'a':[1,2,3],'b':[1,2,3],'c':[1,2,3]}

Disregard the fact that the values for each key are the same. With the FRED data, they will actually be different by virtue of calling different indicators. (The indicator will be used to both name the key and function as the first argument in the web.get_fred_data pull.)

We will build the dictionary in two ways: verbose and compact.

Now we should start thinking about what we might actually do from an analytical standpoint. Let's say we are designating per capita income as our dependent variable. What about regressors? This script is more about working with data in pandas than model specification, so we are going to just grab a few variables that already sit in FRED. Items that could affect per capita income:

  1. Business cycle via unemployment rate proxy (*UR)
  1. Population density (*POP - this is only population, but we can identify the land area)
  1. Government employment (*GOVTN)

Perhaps we want to make the case that health insurance coverage impacts earnings by preventing disruptive losses from derailing earning trajectories of workers. Our weird variable of interest, then, is health insurance coverage percentage (*HICCOVPCT). Note that the data for this variable go back only to 1999.

Given that the input for Panel is just a dictionary that holds DataFrames, we can construct this dictionary any way we like. We will use a strategy that tries to minimize the editing scope if we decide we want to change variables or states.

#Construct list of indicator suffixes (will be joined programmatically with states)
ind_suffix_list=['PCPI','UR','POP','GOVTN','HICCOVPCT']

#Construct state list
state_list=['DC','MD','VA']

#Construct land area list
land_area=[68.3,12407.0,42775.0]

#Create function to generate dictionary
def fred_dict(groups,indicators,start_date,end_date,comvar):
    """Function grabs FRED data and constructs a dictionary:

            'jurisdiction':[DataFrame holding associated variables]

        ***Note that this function aggregates to yearly values via mean().***
    """
    #Create container for group-specific DFs (will be bound to group names in output dict)
    df_list=[]
    i=0
    for group in groups:
        #Create container for within group DFs (for each indicator)
        subdf_list=[]
        #Create container for full indicator names (for use in renaming)
        indic_list=[]
        for indic in indicators:
            #Construct indicator name and throw it in indic_list
            ind=group+indic
            indic_list.append(ind)
            #Call data and throw it into the within group DF list
            subdf_list.append(web.get_data_fred(ind,start_date,end_date))
        #Concatenate to one DF per group
        df=pd.concat(subdf_list,axis=1)
        #Annualize data by averaging over each year
        df2=df.groupby(df.index.year).mean()
        #Create renaming dictionary
        rename_dict=dict(zip(indic_list,comvar))
        #Rename columns
        df3=df2.rename(columns=rename_dict)
        #Add in state land area for use in constructing population density
        df3['area']=land_area[i]
        i+=1
        #Calculate population density
        df3['POP_DENS']=df3['POP']/df3['area']
        #Drop population and area columns
        df3.pop('area')
        df3.pop('POP')
        #Throw group DFs into a values list for binding with keys
        df_list.append(df3)
    #Construct output dictionary
    df_dict=(dict(zip(groups,df_list)))
    return df_dict

Looks like we have a usable DF, and we can make some comparisons. These comparisons would be easier to see graphically, so let's plot the data. Ideally we could have color mapped to the p_value, so we could have a clear idea of the relative significance of each estimate. This is fairly trivial with ggplot2 (in R), but using rmagic is not the subject of this script. Consequently, we will make do with the sometimes clunky matplotlib.

The workaround is to create discrete color mapping in a new variable. Based upon popular notions of statistical significance, we will assign three colors based upon p-value breakpoints of .05 and .1. The darker the color (light: green, medium: blue, dark: purple), the higher the degree of "statistical significance" based upon our p-value metric.

Note that to create this plot, we could not rely on the pandas DataFrame.plot() method. These methods are merely pandas wrappers for matplotlib, so we only gather a subset of the functionality. Usually it's ample, but sometimes we need more...

The results above (superficially) suggest that state-specific factors are quite important in determining the impact of our covariates on per capita income. We, thus, have cause to explore the panel nature of these data.

Note that we will be using our Panel structure for this section instead of indexing down to a single DataFrame. This increases the dimensionality of our inputs. When we ran OLS above, our dependent (y_ols) was a single Series, basically a 1-D array. Our covariates (X_ols) were housed in a DataFrame, a 2-D array. Now that we are performing panel analysis, our dependent (y_panel) will be a DataFrame of the same length as our previous dependent (y_ols). However, it now has three columns, one for each group.

The pandas ols class will recognize the shift in inputs, and alter the way it treats the data. Furthermore, additional options (pool, entity_effects, time_effects, etc.) for panel analysis will become available.

First, let's try to abstract away from specific states, and explore a pooled model. Note that the minor axis in the X Panel object (x_panel) must correspond with the column names in the y DataFrame (y_panel). If this is not the case, trying to run the panel analysis will throw a cryptic error. The one I saw was a math domain error (essentially claiming the instance of a figure value that could not be achieved, like a log operation returning a negative number), and the stack trace pointed to a specific line number rather than associated code (which I moved around when attempting to print values). Save yourself the hassle and just make sure your axes are where they are supposed to be (this is the reason for .swapaxes() call below).
